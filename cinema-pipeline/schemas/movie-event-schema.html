Выбор ключа: ключ партиционирования = user_id. Все события одного пользователя попадают в одну партицию → Kafka гарантирует порядок внутри сессии (VIEW_STARTED → VIEW_PAUSED → VIEW_RESUMED → VIEW_FINISHED). Альтернатива movie_id создала бы «горячие» партиции для популярных фильмов.
